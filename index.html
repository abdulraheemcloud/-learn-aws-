📚 AWS Glossary

Availability Zone (AZ) AZ
One or more discrete data centers with redundant power, networking, and connectivity in an AWS Region. AZs are physically separated to protect against disasters.
Elastic Block Store EBS
Persistent block storage volumes for use with Amazon EC2 instances. EBS volumes are network-attached and persist independently from the life of an instance.
Virtual Private Cloud VPC
Logically isolated section of the AWS cloud where you can launch AWS resources in a virtual network that you define. Complete control over IP addressing, subnets, and routing.
Identity Access Management IAM
Web service for securely controlling access to AWS resources. Enables you to manage users, security credentials, and permissions to access AWS resources.

EC2 & Compute Services
Module 02 • 4 Labs • 8 Tasks • Quiz

📋 Module Overview

Amazon EC2 (Elastic Compute Cloud) provides resizable compute capacity in the cloud. This module covers instance types, AMIs, security groups, key pairs, and pricing models including On-Demand, Reserved, and Spot instances.

🎯 Learning Objectives

✓ Understand EC2 instance families and their use cases (T3, M5, C5, R5, etc.)
✓ Launch and configure EC2 instances with proper security groups
✓ Connect to instances using SSH (Linux) and RDP (Windows)
✓ Understand EC2 pricing models and cost optimization strategies
💡 Pro Tip Always use t3.micro or t2.micro for free tier eligibility when learning. Set up billing alerts to avoid unexpected charges.
🧪
Lab Environment: You will launch a Linux EC2 instance, configure security groups, and connect via SSH. Estimated time: 30 minutes.
01

Launch EC2 Instance

Navigate to EC2 Dashboard and launch a new instance

1. Sign in to AWS Management Console
2. Navigate to EC2 → Instances → Launch Instance
3. Choose Amazon Linux 2 AMI (Free tier eligible)
4. Select t2.micro instance type

AWS CLI Equivalent 
aws ec2 run-instances \
  --image-id ami-0c55b159cbfafe1f0 \
  --count 1 \
  --instance-type t2.micro \
  --key-name MyKeyPair \
  --security-group-ids sg-903004f8 \
  --subnet-id subnet-6e7f829e
02

Configure Security Group

Allow SSH access from your IP address only

⚠️ Security Warning Never open SSH (port 22) to 0.0.0.0/0 (anywhere). Always restrict to your specific IP address or VPN range.

Inbound Rules:
• Type: SSH (22)
• Source: My IP (auto-detects your public IP)
• Description: SSH access from office

03

Connect via SSH

Use your private key to establish secure connection

Terminal 
# Set correct permissions on key file
chmod 400 MyKeyPair.pem

# Connect to instance
ssh -i "MyKeyPair.pem" ec2-user@<public-ip>
✅ Success Check You should see the Amazon Linux 2 welcome message and prompt: [ec2-user@ip-172-31-0-1 ~]$
